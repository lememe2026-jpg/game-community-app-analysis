# AI 辅助工作流与人工判断边界

本文用于说明本项目中 AI / Codex 的辅助范围，以及哪些判断由人工完成。它不是工具宣传，也不用于说明 AI 独立完成项目。

## 项目性质

本项目是一次 AI / Codex 辅助下的个人学习型公开数据分析实践。项目目的不是做商业咨询报告，也不是评价小黑盒、游民星空或相关公司的真实运营能力，而是通过公开数据练习数据清洗、脚本阅读、结果复核、报告表达和公开仓库整理。

## AI / Codex 辅助过的环节

AI / Codex 在本项目中主要作为辅助工具，用于帮助我提高阅读、整理和检查效率：

- 代码阅读：帮助理解 `scripts/` 中清洗、评论、关键词、下载、版本和四源整合脚本的大致用途。
- 脚本解释：辅助解释输出表和图表如何由脚本产生，但不替代人工复核。
- 格式清理：辅助检查 Markdown 结构、文件路径、图表引用和目录关系。
- 检查清单：辅助整理报告检查、公开仓库检查和边界表达检查项。
- 文档草稿：辅助起草 README、说明文档、修订记录和流程记录。
- 口径风险检查：辅助识别“证明增长”“评价公司能力”“官方数据”等不适合公开数据项目的表达。
- 流程记录辅助：帮助把人工修改、复核和公开仓库整理过程记录成可追溯文档。

## 人工负责的环节

以下内容由本人负责最终判断和确认：

- 项目选题与定位：确定项目是个人学习型公开数据分析，而不是商业咨询或行业研究结论。
- 报告手动修改：决定正式报告的章节结构、保留内容、删减内容和最终表达。
- 数据解释边界：判断哪些只能写成公开观察，哪些不能写成因果、效果或真实业务结论。
- Excel 复核：使用表格方式核对关键汇总、月份、峰值、评论数量和关键词分类口径。
- 图表取舍：决定哪些图表进入正式报告，哪些仅作为过程结果。
- 关键结论取舍：对主要观察、限制说明和后续学习方向做最终确认。
- 公开仓库整理：确认公开仓库只保留汇总表、图表、脚本、报告和必要说明文档。

## 不应被误解的地方

本项目不表达以下含义：

- 不是 AI 独立完成项目。
- 不是本人已经熟练掌握所有代码、SQL 或完整数据工程流程。
- 不是用 AI 证明某个增长原因。
- 不是用公开数据评价公司真实运营能力。
- 不是把 AI 输出直接作为最终结论。

更准确的表述是：AI / Codex 参与了代码理解、文档整理、检查清单和口径风险检查，但项目定位、报告修改、数据解释边界、关键结论取舍和最终确认由本人完成。

## 使用 AI 时的复核原则

- 对数字、路径、日期必须回到项目文件核对。
- 对下载、关键词、版本和评论之间的关系，只写观察，不写效果证明。
- 对 TGA 等外部事件只写外部观察变量，不写下载增长原因。
- 对无法确认的信息不写成确定结论。
- 对正式展示内容，优先以 `portfolio/game_community_app_analysis_report.md` 和 PDF 为准。
